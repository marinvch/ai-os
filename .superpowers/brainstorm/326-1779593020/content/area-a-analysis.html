install.sh vs bootstrap.sh Undocumented distinction

bootstrap.sh = remote installer — clones AI OS repo to /tmp, then runs install.sh. Used when you do curl url | bash or when you don't have the repo locally. install.sh = local installer — runs from inside the cloned repo. The distinction is real and important, but zero documentation explains this. First-time contributors are confused about which to use or edit.

→ Propose: Add a header comment to both files + docs/contributing.md section explaining the two-script architecture.
