A2A in AI OS — What's the core goal?

AI OS currently generates MCP tools, agents, skills & instructions for repos. Where does A2A fit?

A

Make AI OS agents discoverable

Generate an AgentCard (/.well-known/agent-card.json) for each repo so external AI agents can discover and call the repo's capabilities via the A2A protocol.

B

Turn the MCP server into an A2A server

Expose AI OS's MCP server as an A2A-compatible HTTP endpoint — other agents (Google, Claude, GPT-based) can delegate tasks (search code, query memory, introspect stack) to it.

C

Generate A2A config for the target repo

AI OS scans the repo and auto-generates A2A configuration — AgentCard, skill definitions, task schemas — so the repo's own agents become A2A-interoperable out of the box.

D

AI OS as an A2A orchestrator

AI OS acts as a master orchestrator that routes tasks across multiple A2A-compatible agents (code agent, test agent, deploy agent) coordinated by the repo context it generates.
