Architecture: AI OS Orchestrator

How a developer task flows through the system

👤 Developer
"implement a Stripe checkout page"
↓ chat
🧠 orchestrator.agent.md (generated by AI OS)
• Knows all sub-agents via agents.json
• Calls list_agents() → finds "stripe-agent", "nextjs-agent"
• Calls delegate_to_agent("stripe-agent", task)
• Synthesizes result → responds to developer
💳 stripe-agent.md
payments, checkout, webhooks
⚡ nextjs-agent.md
pages, routing, SSR/RSC
🗄️ prisma-agent.md
schema, migrations, queries
🔧 MCP Server (extended)
list_agents()
Returns all agents + capabilities from agents.json
delegate_to_agent(name, task)
Routes task to sub-agent, returns result
📋 .github/ai-os/agents.json (new artifact, A2A-inspired AgentCard registry)
{ "stripe-agent": { "capabilities": ["payments","checkout"], "triggers": ["stripe","payment"] }, ... }
